What is REST?

REST - Representation state tranfer.


It is a style of software architecture, which has became a predominant web API design model.

About REST

REST architetural design consists of clients and servers.


They exchange/transfer the respresetations of resources.


A resource is essentially any meaningful concept.


A representation of a resource is typically a document that captures the current or intended state of a resource.

Guiding Principles

  • Identification/existence of Resources
  • Manipulation of resources through these representations
  • Communication via a standardized interface (e.g. HTTP)
  • Layered System
  • Stateless

REST in Web

A RESTful web API is a web API implementated using HTTP and REST principles.


It is a collection of resources with :

  • Operations (HTTP methods).
  • A base URI and sensible resource names.
  • The internet media type of the data (JSON and XML).

Operations

  • GET : retrieve a resource/resources. 
                    
"GET": /me/friends
"GET": /me/friends/4802387
  • PUT : update an existing resource. 
                    
"PUT": /me/friends/4802387
  • POST : create of new resources 
                    
"POST": /me/friends
  • DELETE : delete a resource. 
                    
"DELETE": /me/friends/4802387

Resource naming

A RESTful API essentially is a colleciton of URIs.


  • Well named resources ==> intuitive, easy to use.
  • Poorly designed ones ==> it can be very confusing.

Tips

  • Each resource has its own address or URIs.
  • URIs shall adequately describes the resource.
  • URIs should follow a predictable, hierarchical structure to enhance understandability and usability
  • URIs should convey meaning to the consumer.


Examples

              
  "GET" : statuses/retweets/:id
  "GET" : statuses/show/:id
  "POST" : statuses/destroy/:id
  "POST" : statuses/update/
  "POST" : statuses/retweet/:id

Data Type

Favor JSON support, but offering both JSON and XML are very helpful.


Ideally, let consumers switch between them by just changing an extension from .xml to .json.

In Rails.

Rails embraces RESTful resource routing.


The route provides a mapping between HTTP verbs and URLs to controller actions.


How to create route.

              
$ rake routes

Single Resource

              
REST request path    | Rails action name | Description 
---------------------|-------------------|------------- 
GET /profile/new     | new               | Render a form for creating the profile 
POST /profile        | create            | Create a the profile from the received data 
GET /profile         | show              | Render a the profile 
GET /profile/edit    | edit              | Render a form for editing the profile 
PUT /profile         | update            | Update the profile based on the received data 
DELETE /profile      | destroy           | Destroy the profile

Multiple Resources

              
REST request path    | Rails action name | Description 
---------------------|-------------------|------------- 
GET /posts           | index             | Render a list of all post 
GET /posts/new       | new               | Render a form for creating a single new post 
POST /posts          | create            | Create a single new post from the received data 
GET /posts/1         | show              | Render a single existing post 
GET /posts/1/edit    | edit              | Render a form for editing a single existing post 
PUT /posts/1         | update            | Update a single existing post based on the received data 
DELETE /posts/1      | destroy           | Destroy a single existing post
